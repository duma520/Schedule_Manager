## 项目概述

### 1.1 什么是排班表管理系统？
排班表管理系统是一款专为各类组织设计的可视化排班工具，适用于医院、工厂、餐厅、零售店、学校、安保公司等多种场景。系统采用直观的日历界面和列表视图，帮助管理人员高效安排员工工作时间。

### 1.2 适用人群
- **基层员工**：查看自己的排班安排
- **班组长**：管理小团队的工作分配
- **人力资源**：统筹全公司的人力资源调配
- **企业管理者**：监控整体人力成本和工作安排
- **自由职业者**：管理自己的工作时间表

### 1.3 核心优势
- 多用户支持，数据隔离安全
- 直观的月历和列表双视图
- 灵活的班次自定义功能
- 部门分类管理
- 跨平台兼容性（Windows/macOS/Linux）

## 系统功能

### 2.1 主要功能模块
1. **用户管理**：多用户系统，支持密码保护
2. **排班管理**：添加、编辑、删除排班记录
3. **视图切换**：月历视图和列表视图
4. **部门管理**：自定义部门分类
5. **班次管理**：预设+自定义班次类型
6. **数据过滤**：按日期、部门、姓名筛选

### 2.2 技术特点
- 基于Python 3和PyQt5开发
- SQLite数据库存储
- 响应式界面设计
- 马卡龙色系可视化区分

## 安装与运行

### 3.1 系统要求
- 操作系统：Windows 7+/macOS 10.12+/Linux（需图形界面）
- Python版本：3.6或更高
- 内存：至少2GB RAM
- 磁盘空间：50MB可用空间

### 3.2 安装步骤
**方法一：直接运行（推荐非技术人员）**
1. 从发布页面下载打包好的可执行文件
2. 双击运行即可

**方法二：源代码运行（适合开发者）**
```bash
git clone https://github.com/duma520/schedule-manager.git
cd schedule-manager
pip install -r requirements.txt
python Schedule_Manager.py
```

### 3.3 首次运行
1. 系统会自动创建必要的数据库文件
2. 首次使用建议创建管理员账户
3. 系统内置了常见部门和班次模板

## 用户管理

### 4.1 用户类型
- **无密码用户**：快速登录，适合个人使用
- **密码保护用户**：团队使用，保障数据安全

### 4.2 用户操作指南
**注册新用户：**
1. 点击登录界面的"注册新用户"
2. 输入用户名（必填）
3. 可选设置密码
4. 点击确定完成注册

**删除用户：**
1. 需要输入正确密码验证
2. 系统会同时删除该用户的所有排班数据
3. 操作不可逆，请谨慎执行

**切换用户：**
1. 主界面右上角点击"切换用户"按钮
2. 重新登录其他账户
3. 注意：未保存的数据会丢失

## 基础使用指南

### 5.1 快速入门
**场景：咖啡店店长安排下周班表**
1. 登录系统
2. 切换到目标月份
3. 双击日期格子添加排班
4. 输入员工姓名：张三
5. 选择部门："咖啡师"
6. 选择班次："早班(8:00-16:00)"
7. 点击确定保存

### 5.2 月历视图操作
- **导航**：使用"上个月"/"下个月"按钮切换
- **添加排班**：双击日期格子
- **查看详情**：单击日期格子显示当天所有排班
- **右键菜单**：在日期上右键可快速编辑/删除

### 5.3 列表视图操作
- **筛选数据**：使用顶部过滤条件
- **批量操作**：支持多选后批量删除
- **排序**：点击列标题可按该列排序

## 高级功能

### 6.1 自定义部门
**场景：医院新增专科门诊**
1. 在排班对话框点击"自定义部门"
2. 输入新部门名称："心脏外科"
3. 确定后该部门即出现在下拉选项中

### 6.2 特殊班次设置
**场景：工厂设置弹性工作制**
1. 点击"自定义时间段"
2. 输入班次名称："弹性班A"
3. 设置时间范围：9:00-15:00（核心工作时间）
4. 保存后即可选择该班次

### 6.3 数据导出
虽然当前版本未内置导出功能，但可通过以下方式导出数据：
1. 使用SQLite浏览器直接打开用户数据库文件
2. 执行SQL查询导出CSV：
```sql
.headers on
.mode csv
.output schedule.csv
SELECT * FROM schedules;
```

## 专业应用场景

### 7.1 医疗机构排班
**特点**：
- 三班倒（早/中/晚）
- 跨24小时连续排班
- 需要处理节假日特别安排

**最佳实践**：
1. 创建专门部门："急诊科"、"住院部"等
2. 设置模糊班次："待命班"（无固定时间）
3. 使用颜色区分不同职称人员

### 7.2 零售业排班
**特点**：
- 周末高峰需要更多人手
- 兼职人员多
- 班次变化频繁

**技巧**：
1. 利用备注字段记录特殊要求
2. 提前一个月规划好班表
3. 使用部门过滤快速查看各区域安排

### 7.3 工厂生产排班
**特点**：
- 四班三运转
- 需要考虑交接班时间
- 机器维护时段安排

**解决方案**：
1. 创建自定义班次："早班(06:00-14:00)+交接"
2. 使用职位字段标注"组长"等特殊角色
3. 每月底导出数据做工时统计

## 数据管理与安全

### 8.1 数据存储位置
- 用户数据：`users.db`（所有用户共享）
- 排班数据：`user_用户名.db`（每个用户独立）

### 8.2 备份策略
1. 定期复制数据库文件到外部存储
2. 建议备份频率：
   - 小型团队：每周一次
   - 大型机构：每日一次

### 8.3 数据恢复
1. 关闭系统
2. 用备份的数据库文件替换损坏的文件
3. 重新启动系统

## 常见问题解答

### 9.1 基础问题
**Q：忘记密码怎么办？**
A：系统设计为无密码找回功能，需删除重建用户（会丢失原有数据）

**Q：能多人同时使用吗？**
A：可以，但同一账号不能同时登录

### 9.2 技术问题
**Q：数据库损坏如何处理？**
A：尝试使用SQLite修复工具，或从备份恢复

**Q：如何迁移到新电脑？**
A：复制整个程序目录和所有.db文件即可

### 9.3 功能需求
**Q：能否增加排班冲突检测？**
A：此功能已在开发路线图中

**Q：是否支持移动端？**
A：当前仅限桌面端，未来可能开发移动版本
